## Git 運用ルール

### 基本方針
- **コードを変更するたびに必ずGitHubにプッシュする**
- `main` ブランチへの直接プッシュは原則禁止。作業は feature ブランチで行う
- コミットメッセージは変更内容が明確にわかるよう日本語または英語で記述する

### ワークフロー
1. 作業開始前に `git pull` で最新を取得する
2. 変更をステージング: `git add <ファイル名>`（`git add .` は機密ファイルの混入を避けるため慎重に使う）
3. コミット: `git commit -m "変更内容の説明"`
4. **即時プッシュ: `git push origin <ブランチ名>`**
5. 機能が完成したらPRを作成してレビュー後にマージする

### コミットの粒度
- 1つの論理的な変更につき1コミットを目安にする
- WIPコミットは許容するが、PRマージ前にsquashまたはrebaseで整理する

### 禁止事項
- `git push --force` は原則禁止（`--force-with-lease` は許容）
- `--no-verify` によるフックのスキップは禁止
- `.env` や認証情報を含むファイルのコミットは厳禁
